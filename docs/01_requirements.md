# MDE — 요구사항 정의

## 기능 요구사항

### FR-01: 텍스트 기반 음악 방향성 분석
- 사용자는 자유형식 텍스트(10~500자)를 입력할 수 있다
- 시스템은 MusicProfile(9개 필드) + 4개 방향성 설명을 반환해야 한다
- 응답 시간은 10초 이내여야 한다 (Gemini 2회 호출 기준)

### FR-02: 이미지 기반 음악 방향성 분석
- 사용자는 이미지(JPEG, PNG, WebP, GIF, 최대 10MB)를 업로드할 수 있다
- 시스템은 이미지를 Gemini Vision으로 분석해 한국어 감성 묘사로 변환한다
- 변환된 텍스트로 FR-01과 동일한 분석을 수행한다
- 이미지 설명(imageDescription)을 결과에 함께 반환한다

### FR-03: 사용자 인증
- 이메일/비밀번호 기반 회원가입 및 로그인
- JWT 기반 인증 (HTTP-only 쿠키, 7일 만료)
- 비로그인 사용자도 분석 가능 (결과는 저장되나 히스토리 조회 불가)

### FR-04: 분석 히스토리
- 로그인 사용자의 분석 결과를 DB에 저장
- 히스토리 목록 조회, 상세 조회, 삭제, 텍스트 검색 제공

### FR-05: 공유 링크
- 분석 결과를 누구나 접근 가능한 링크로 공유
- 공유 링크 조회 시 view_count 증가

### FR-06: 관리자 기능
- 전체 사용자 수, 세션 수, 공유 수, 조회 수 통계
- 최근 공유 목록 조회

## 비기능 요구사항

### NFR-01: 보안
- 비밀번호 bcrypt 해시 저장 (passlib + bcrypt==4.2.1)
- JWT HS256, 동일 시크릿을 프론트/백엔드에서 공유
- HTTP-only 쿠키로 XSS 토큰 탈취 방지
- Next.js 프록시 패턴으로 쿠키 도메인 문제 없이 전달

### NFR-02: 성능
- 텍스트 분석 rate limit: IP당 200회/분
- 이미지 분석 rate limit: IP당 10회/분
- X-Forwarded-For 헤더로 실제 클라이언트 IP 추출

### NFR-03: 확장성
- DATABASE_URL 변경으로 SQLite → PostgreSQL 전환 가능
- LLM_PROVIDER 변경으로 다른 모델 전환 가능 구조

### NFR-04: 가용성
- Gemini API 429 응답 시 사용자에게 명확한 한국어 에러 메시지 반환
- 방향성 설명 생성 실패 시 fallback 텍스트로 대체

## 입력 제약

| 항목 | 제약 |
|------|------|
| 텍스트 최소 길이 | 10자 |
| 텍스트 최대 길이 | 500자 |
| 이미지 형식 | JPEG, PNG, WebP, GIF |
| 이미지 최대 크기 | 10MB |
| 텍스트 분석 rate limit | 200회/분 (IP별) |
| 이미지 분석 rate limit | 10회/분 (IP별) |
