# 실험 001 — LLM 출력 일관성 개선

## 실험 목적

동일한 입력에 대해 반복 실행 시 핵심 속성(emotion, genre)의 일치율을 높이는 방법 탐색.

## 문제

초기 구현에서 `temperature=0.7`을 사용했을 때:
- 같은 입력에서 `energy`가 `low`와 `medium`을 번갈아 반환
- `genre`에서 핵심 장르가 누락되거나 과도하게 세분화됨
- `atmosphere` 키워드가 매번 다른 단어 선택

## 실험 설계

| 조건 | temperature | 실행 횟수 | 측정 지표 |
|------|------------|---------|---------|
| A | 0.1 | 20 | emotion 겹침률, genre 겹침률 |
| B | 0.3 | 20 | emotion 겹침률, genre 겹침률 |
| C | 0.5 | 20 | emotion 겹침률, genre 겹침률 |
| D | 0.7 | 20 | emotion 겹침률, genre 겹침률 |

## 테스트 입력

```
"새벽 3시, 비 오는 날, 혼자 운전하는 느낌"
```

## 결과

| 조건 | emotion 겹침률 | genre 겹침률 | energy 일치율 | 비고 |
|------|-------------|------------|------------|------|
| A (0.1) | 91% | 89% | 100% | 너무 획일적, 창의성 부족 |
| B (0.3) | 84% | 81% | 95% | 안정적, 적절한 다양성 |
| C (0.5) | 73% | 68% | 87% | 분산 증가 |
| D (0.7) | 58% | 54% | 76% | 과도한 분산 |

## 결론

`temperature=0.3`을 MusicProfile 생성에 채택.

추가로:
- `energy`, `tempo_feel`의 허용 값을 프롬프트에 명시 (`low | medium | high`)
- `genre` 후보 목록을 프롬프트에 제공해 일반적이지 않은 장르명 방지
- JSON 스키마를 시스템 프롬프트에 명시해 필드 누락 최소화

## 적용 결과

프롬프트 개선 + temperature=0.3 적용 후:
- JSON 파싱 실패율: 8% → 3% 감소
- energy 일치율: 76% → 95% 향상
- 핵심 genre 포함율: 54% → 81% 향상
